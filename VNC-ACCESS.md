# Вход через VNC (Beget) и восстановление SSH

## Почему VNC «постоянно просит логин»

Это **нормально** — это экран входа в Linux, не ошибка панели.

На чёрном экране должно быть что-то вроде:

```
SEOLES login: _
```

### Шаг за шагом

1. В панели Beget: **Облако → SEOLES → Терминал → VNC** (откроется консоль в браузере).
2. Кликните **внутрь** чёрного окна (чтобы фокус был на консоли).
3. Наберите **`root`** → Enter.
4. Строка **`Password:`** — введите пароль **с клавиатуры** (символы **не отображаются**) → Enter.

### Если пароль не принимает

1. Beget → **SEOLES** → блок **«Реквизиты доступа»** — скопируйте пароль **оттуда** (может отличаться от того, что вы помните).
2. Или **смените пароль root** в панели (кнопка смены пароля у VPS) → подождите 1–2 мин → повторите вход в VNC.
3. Раскладка **EN** (латиница). Пароль содержит `*` и `%` — на русской раскладке они не те клавиши.
4. Вводите пароль **вручную**, не через буфер обмена (VNC иногда ломает вставку).

### После успешного входа

Вы увидите приглашение:

```
root@SEOLES:~#
```

Скопируйте и выполните **одной вставкой** (или построчно):

```bash
curl -fsSL https://raw.githubusercontent.com/placeholder 2>/dev/null || true
bash /opt/seoles/infra/scripts/vnc-fix-ssh.sh
```

Пока проект не на сервере — вставьте содержимое файла `infra/scripts/vnc-fix-ssh.sh` с Mac:

```bash
cat ~/Desktop/seoles/infra/scripts/vnc-fix-ssh.sh
```

Скопируйте вывод и вставьте в VNC.

Или минимум вручную:

```bash
systemctl restart ssh || systemctl restart sshd
ufw allow 22/tcp; ufw allow 80/tcp; ufw allow 443/tcp; ufw disable
ss -tlnp | grep 22
```

## Настройки сети в Beget

Слева в меню облака: **«Настройки сети»** — если есть firewall / security group, разрешите **входящие TCP 22, 80, 443** на IP `2.56.240.192`.

## Проверка с Mac

```bash
ssh -v root@2.56.240.192
```

Успех: строка `Authenticated to` или запрос пароля без `Connection timed out`.

## Дальше — деплой SEOLES

Когда SSH заработает:

```bash
cd ~/Desktop/seoles
./infra/scripts/generate-env.sh | tee .env
./infra/scripts/deploy.sh root@2.56.240.192
```

Сайт: http://2.56.240.192
